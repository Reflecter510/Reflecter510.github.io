寻找两个正序数组的中位数

二分查找
algorithm
发布日期:   2020-09-20
更新日期:   2020-09-22
文章字数:   1.3k
阅读时长:   5 分

问题描述

给定两个大小为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。

请你找出这两个正序数组的中位数，并且要求算法的时间复杂度为 O(log(m + n))。

你可以假设 nums1 和 nums2 不会同时为空。

 

示例 1:

nums1 = [1, 3]
nums2 = [2]

则中位数是 2.0

示例 2:

nums1 = [1, 2]
nums2 = [3, 4]

则中位数是 （2 + 3）/2 = 2.5

二分查找

暴力解法

  • 最直观的思路是将这个两个有序数组进行合并，类似于归并排序里的合并操作。
  • 另一个思路是使用双指针，每次将较小的那个指针往后移动一位，直到第(len1+len2)/2个元素（当然需要按数组总长度的奇偶数来谈论），这样只需要O(1)的空间复杂度。

但是这两种思路都是O(m+n)的时间复杂度，不足以满足题目要求。

进一步优化，二分思路
二分查找在第二种思路上更进一步，我们不需要一个一个地往后移指针。因为我们知道两个数组的长度，所以求中位数的问题可以转换为求在合并后的有序数组中的第 k 个元素的问题（不过并不需要真的合并）。

  • 对第 k 个元素进行二分，每次找这两个有序数组中第 k/2 个元素进行比较。
  • 较小元素的那个数组前 k/2 个元素肯定不是我们要求的第 k 个元素，因此可以抛弃这 k/2 个元素。
  • 抛弃元素后数组又是一个规模更小的有序数组，并且要寻找的第 k 个元素也更靠前了。返回第一步继续求解，直到问题的规模可以直接求解，也就是下面说的特殊情况。

但是有可能出现数组长度小于 k/2，此时应取数组长度作为抛弃长度，因此改进一下：

  • 使用index1和index2来表示分别表示数组的起始下标，下标小于index的元素均视为被抛弃的。
  1. newIndex 为 index+k/2-1 与 len-1 的最小值
  2. 当A[newIndex1] <= B[newIndex2]时，index1 = newIndex1+1，k -= newIndex1-index1+1
  3. 否则相反
  4. 重复 1 和 2 直到遇到特殊情况。

特殊情况（直接求解）：

  • k=1，说明取第一个元素，那么在两个数组首部取最小值即可。
  • index == len，说明数组为空，在另一个数组取第 k 个元素即可。
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        int len1 = nums1.size(), len2 = nums2.size();
        int k = (len1 + len2) / 2;

        if((len1+len2)%2 != 0){
            return 0.0 + getKthElement(nums1, nums2, k+1);
        }else{
            return (getKthElement(nums1, nums2, k) + getKthElement(nums1, nums2, k+1))/2.0;
        }
    }

    int getKthElement(vector<int> &num1, vector<int> &num2, int k){
        int len1 = num1.size(), len2 = num2.size();
        int index1 = 0, index2 = 0;

        while(true){
            //特殊情况
            if(index1 == len1){   //num1数组为空，那么直接在num2数组求即可
                return num2[index2+k-1];
            }
            if(index2 == len2){   //num2数组为空，那么直接在num1数组求即可
                return num1[index1+k-1];
            }
            if(k == 1){           //k=1时，直接返回两个数组第一个元素的较小者即可
                return min(num1[index1], num2[index2]);
            }
            //正常情况，需要注意有时候数组长度小于 k/2，此时应取数组长度作为抛弃长度
            int newIndex1 = index1 + min(k/2-1, len1-1-index1);
            int newIndex2 = index2 + min(k/2-1, len2-1-index2);
            if(num1[newIndex1] <= num2[newIndex2]){
                k -= newIndex1-index1+1;
                index1  = newIndex1+1;
            }else{
                k -= newIndex2-index2+1;
                index2  = newIndex2+1;
            }
        }
    }
};

复杂度分析

  • 时间复杂度：O(log(m+n))，其中 m 和 n 分别是数组 nums1 和 nums2 的长度。初始时有 k=(m+n)/2 或 k=(m+n)/2+1，每一轮循环可以将查找范围减少一半，因此时间复杂度是 O(log(m+n))。
  • 空间复杂度：O(1)。

划分数组（更低复杂度的解法）

官方题解讲得十分细致。

需要注意二分时 i 的取值，以及数组越界这种特殊情况的讨论。

  • left = 0，right = len1，i = (left+right)/2;
  • 对于 i=0、i=m、j=0、j=n 这样的临界条件，我们只需要规定A[−1]=B[−1]=−∞，A[m]=B[n]=∞ 即可。这也是比较直观的：当一个数组不出现在前一部分时，对应的值为负无穷，就不会对前一部分的最大值产生影响；当一个数组不出现在后一部分时，对应的值为正无穷，就不会对后一部分的最小值产生影响。
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        int len1 = nums1.size(), len2 = nums2.size();

        if(len1 > len2)
            return findMedianSortedArrays(nums2, nums1);

        int mid1 = 0, mid2 = 0;
        int left = 0, right = len1;

        //二分找最大的 i 使得 A[i-1] < B[j]
        // i+j = m-i+n-j +1?
        // j = (m+n+1)/2-i;
        while(left <= right){
            int i = (left+right)/2;
            int j = (len1+len2+1)/2-i;

            int vi_pre = i == 0? INT_MIN : nums1[i-1];
            int vi = i == len1? INT_MAX : nums1[i];
            int vj_pre = j == 0? INT_MIN : nums2[j-1];
            int vj = j == len2? INT_MAX : nums2[j];

            if(vi_pre <= vj){
                mid1 = max(vi_pre, vj_pre);
                mid2 = min(vi, vj);
                left = i+1;
            }else{
                right = i-1;
            }
        }

        if((len1+len2) %2 == 1){
            return mid1;
        }else{
            return (mid1+mid2)/2.0;
        }
    }
};
